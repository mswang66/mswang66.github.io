mariadb集群zookeeper管理逻辑

发表于 2018-02-02

启动之前逻辑

  1. 先获取锁
  2. 判断当前是否有服务在线

  3. 如果有服务在线，则直接 slave 形式启动，并在启动结束之后放弃锁

  4. 如果没有服务在线,
    3.1. 查看本地seq
    3.2. 如果seq > 0

    master形式 启动
放弃锁

    3.3. 如果seq = -1

    3.3.1 查看Zookeeper上是否存在last
3.3.2 如果不存在 
      直接启动 master形式
3.3.3 如果存在(可能是多个)
      判断是否属于这2个
      如果是，则master启动
      如果不是， 则放弃锁，并重新获取申请锁

启动逻辑

  1. run command
  2. 监控服务是否正常

突然挂了

一、mysql挂了，容器没挂

  1. 删除本地对应的临时节点 (不用手动删除，zk close即可)
  2. 删除本地对应的永久节点
    这里需要判断当前节点是否是最后一个节点，如果是，则不删除本地的永久节点

二、整个容器挂了

do nothing
